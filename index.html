Database Setup

First, we need to create our database. Think of a database like a filing cabinet ‚Äì it holds all our tables (folders) that contain our data (files).

‚ö†Ô∏è

Before You Begin

Make sure you have SQL Server Management Studio (SSMS) installed and you're connected to your SQL Server instance.

  1. Switch to Master Database

    We start in the "master" database ‚Äì it's like the control room where we create new databases.

    SQL 
    USE master;
GO

  2. Remove Old Database (If Exists)

    If you're re-running this tutorial, we first delete any old version to start fresh.

    SQL 
    -- Check if database exists and remove it
IF EXISTS (SELECT name FROM sys.databases 
           WHERE name = N'DB_LastNameFirstName01')
BEGIN
    -- Kick out any users connected to it
    ALTER DATABASE [DB_LastNameFirstName01] 
        SET SINGLE_USER WITH ROLLBACK IMMEDIATE;
    
    -- Delete the database
    DROP DATABASE [DB_LastNameFirstName01];
END
GO

  3. Create New Database

    Now we create our brand new database and switch to using it.

    SQL 
    -- Create fresh database
CREATE DATABASE [DB_LastNameFirstName01];
GO

-- Switch to our new database
USE [DB_LastNameFirstName01];
GO
    üí°

    Naming Convention

    Replace "LastNameFirstName" with your actual name. Example: DB_SmithJohn01

Creating Tables

Now comes the main part ‚Äì creating our 8 tables. We'll create them in a specific order because some tables depend on others (foreign key relationships).

üìå

Order Matters!

We create "parent" tables first (like Categories), then "child" tables (like Products that belong to categories).

üè∑Ô∏è

TBL_PRODUCT_CATEGORIES

Groups products into categories like Electronics, Clothing, etc.

SQL 
CREATE TABLE TBL_PRODUCT_CATEGORIES
(
    -- Auto-incrementing ID starting at 1
    CATEGORY_ID     BIGINT IDENTITY(1,1) NOT NULL,
    
    -- Category name up to 100 characters
    CATEGORY_NAME   VARCHAR(100) NOT NULL,
    
    -- Make CATEGORY_ID the primary key
    CONSTRAINT PK_PRODUCT_CATEGORIES 
        PRIMARY KEY (CATEGORY_ID)
);
GO

TBL_WAREHOUSES

Physical locations where products are stored

SQL 
CREATE TABLE TBL_WAREHOUSES
(
    WAREHOUSE_ID    BIGINT IDENTITY(1,1) NOT NULL,
    WAREHOUSE_NAME  VARCHAR(100) NOT NULL,
    
    CONSTRAINT PK_WAREHOUSES 
        PRIMARY KEY (WAREHOUSE_ID)

TBL_EMPLOYEES

Staff who process and manage orders

Special Note

TBL_PRODUCTS

Items available for sale (links to Categories)

SQL 
CREATE TABLE TBL_PRODUCTS
(
    PRODUCT_ID      BIGINT IDENTITY(1,1) NOT NULL,
    PRODUCT_NAME    VARCHAR(200) NOT NULL,
    DESCRIPTION     VARCHAR(500) NULL,
    STANDARD_COST   DECIMAL(5,2) NOT NULL,   -- Max: 999.99
    LIST_PRICE      DECIMAL(5,2) NOT NULL,   -- Max: 999.99
    CATEGORY_ID     BIGINT NOT NULL,
    
    CONSTRAINT PK_PRODUCTS 
        PRIMARY KEY (PRODUCT_ID),
    
    -- Links to Categories table
    CONSTRAINT FK_PRODUCTS_CATEGORY 
        FOREIGN KEY (CATEGORY_ID) 
        REFERENCES TBL_PRODUCT_CATEGORIES(CATEGORY_ID)
);
GO

TBL_ORDERS

Customer purchase orders (links to Customers & Employees)

SQL 
CREATE TABLE TBL_ORDERS
(
    ORDER_ID        BIGINT IDENTITY(1,1) NOT NULL,
    CUSTOMER_ID     BIGINT NOT NULL,
    STATUS          VARCHAR(50) NOT NULL,
    EMPLOYEE_ID     BIGINT NOT NULL,
    ORDER_DATE      VARCHAR(20) NOT NULL,        -- SPEC: VARCHAR
    
    CONSTRAINT PK_ORDERS 
        PRIMARY KEY (ORDER_ID),
    CONSTRAINT FK_ORDERS_CUSTOMER 
        FOREIGN KEY (CUSTOMER_ID) 
        REFERENCES TBL_CUSTOMERS(CUSTOMER_ID),
    CONSTRAINT FK_ORDERS_EMPLOYEE 
        FOREIGN KEY (EMPLOYEE_ID) 
        REFERENCES TBL_EMPLOYEES(EMPLOYEE_ID)
);
GO
üõí

TBL_ITEMS

Individual line items within each order

SQL 
CREATE TABLE TBL_ITEMS
(
    ITEM_ID         BIGINT IDENTITY(1,1) NOT NULL,
    ORDER_ID        BIGINT NOT NULL,
    PRODUCT_ID      BIGINT NOT NULL,
    QUANTITY        INT NOT NULL,
    PRICE           DECIMAL(5,2) NOT NULL,       -- SPEC: PRICE column
    
    CONSTRAINT PK_ITEMS 
        PRIMARY KEY (ITEM_ID),
    CONSTRAINT FK_ITEMS_ORDER 
        FOREIGN KEY (ORDER_ID) 
        REFERENCES TBL_ORDERS(ORDER_ID),
    CONSTRAINT FK_ITEMS_PRODUCT 
        FOREIGN KEY (PRODUCT_ID) 
        REFERENCES TBL_PRODUCTS(PRODUCT_ID)
);
GO
üìä

TBL_INVENTORIES

Stock levels for products in each warehouse (composite key)

üìå

Composite Primary Key

This table uses TWO columns together as the primary key. This means the same product can exist in multiple warehouses.

SQL 
CREATE TABLE TBL_INVENTORIES
(
    PRODUCT_ID      BIGINT NOT NULL,
    WAREHOUSE_ID    BIGINT NOT NULL,
    QUANTITY        INT NOT NULL DEFAULT 0,
    
    -- Composite Primary Key (both columns together)
    CONSTRAINT PK_INVENTORIES 
        PRIMARY KEY (PRODUCT_ID, WAREHOUSE_ID),
    CONSTRAINT FK_INVENTORIES_PRODUCT 
        FOREIGN KEY (PRODUCT_ID) 
        REFERENCES TBL_PRODUCTS(PRODUCT_ID),
    CONSTRAINT FK_INVENTORIES_WAREHOUSE 
        FOREIGN KEY (WAREHOUSE_ID) 
        REFERENCES TBL_WAREHOUSES(WAREHOUSE_ID)
);
GO

Let's make sure everything worked correctly! These queries will show you all your tables, columns, relationships, and data counts.

View All Tables

Shows all 8 tables in your database

SQL 
SELECT TABLE_NAME 
FROM INFORMATION_SCHEMA.TABLES 
WHERE TABLE_TYPE = 'BASE TABLE' 
ORDER BY TABLE_NAME;
